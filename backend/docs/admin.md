# Django Admin: роли и разрешения

## Роли пользователей

Роль задаётся в поле `role` модели `User` и назначается через Django-админку суперпользователем.

| Значение `role` | Название | Описание |
|---|---|---|
| *(пусто)* | — | Обычный пользователь, нет доступа в админку |
| `admin` | Администратор | Полный доступ ко всем разделам |
| `hall_manager` | Менеджер зала | Только бронирования столов и записи на мероприятия |
| `content_manager` | Контент-менеджер | Меню, мероприятия, новости, устройства FCM |

Суперпользователь (`is_superuser=True`) имеет полный доступ независимо от `role`.

## Матрица разрешений

| Раздел | Администратор | Менеджер зала | Контент-менеджер |
|---|---|---|---|
| `User` (пользователи) | ✅ полный | ❌ | ❌ |
| `TableBooking` (брони) | ✅ полный | 👁️ просмотр + изменение статуса | ❌ |
| `EventReservation` (записи на события) | ✅ полный | 👁️ просмотр + изменение | ❌ |
| `Event` (мероприятия) | ✅ полный | ❌ | ✅ полный |
| `News` (новости) | ✅ полный | ❌ | ✅ полный |
| `Category`, `Tag`, `Allergen`, `Dish` | ✅ полный | ❌ | ✅ полный |
| `UserDevice` (FCM-устройства) | ✅ полный | ❌ | 👁️ просмотр |

## Как назначить роль

1. Войти в Django-админку как суперпользователь.
2. Перейти в раздел **Users → Пользователи**.
3. Открыть нужного пользователя.
4. Убедиться, что `is_staff = True` (без этого пользователь не попадёт в админку).
5. Выбрать нужную роль в поле **Роль**.

## Техническая реализация

Разрешения проверяются через переопределение методов `has_view_permission`, `has_add_permission`, `has_change_permission`, `has_delete_permission` в каждом `ModelAdmin`.

Файлы:
- `apps/users/admin.py` — `AdminOnlyMixin` + `UserAdmin`
- `apps/bookings/admin.py` — `TableBookingAdmin` (`hall_manager` + `admin`)
- `apps/events/admin.py` — `EventAdmin`, `NewsAdmin` (content), `EventReservationAdmin` (hall)
- `apps/menu/admin.py` — `ContentManagerMixin` + все модели меню
- `apps/notifications/admin.py` — `UserDeviceAdmin` (content + admin)

## Важные нюансы

- Если у пользователя выставлен `is_staff=True`, но `role` пустой, он может войти в интерфейс, но не увидит ни одного раздела.
- Менеджер зала **не может** создавать или удалять брони — только просматривать и менять статус через Staff API (`GET/PATCH /api/v1/bookings/staff/`) или Django-админку.
- Контент-менеджер **не видит** пользователей и их данные — только контентные разделы.

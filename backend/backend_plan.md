Roadmap (Тикеты и шаги разработки)Разбиваю процесс на логические спринты. Каждая задача должна закрываться PR (Pull Request) и проверяться.

Эпик 1: Инфраструктура и база
1.1. Инициализация проекта: Создание репозитория, настройка .gitignore, инициализация чистого Django проекта.
1.2. Контейнеризация: Написание Dockerfile и docker-compose.yml (включая контейнеры для PostgreSQL и Redis).
1.3. Настройка окружения: Настройка переменных окружения (django-environ) и разделение настроек на base.py, dev.py, prod.py.
1.4. Инициализация S3: Настройка django-storages для работы с медиа-файлами через S3-совместимое хранилище.

Эпик 2: Пользователи и Авторизация (Только номер телефона)
2.1. Кастомная модель пользователя: Создание модели User, где логином является номер телефона (без email и пароля). Накатывание первой миграции.  
2.2. Настройка JWT: Интеграция djangorestframework-simplejwt.
2.3. Интеграция SMS-шлюза: Написание сервиса для генерации OTP, сохранения в Redis (TTL 3 минуты) и отправки через внешний API.  
2.4. Endpoints авторизации: Создание API для request_sms и verify_sms (отдает JWT токены).
2.5. Профиль: API для просмотра и изменения имени пользователя и настроек уведомлений.  

Эпик 3: Каталог меню
3.1. Модели меню: Создание моделей Category, Tag, Allergen, Dish. Настройка полей для загрузки видео и фото. 
3.2. Админка меню: Настройка отображения меню в Django Admin (фильтры, поиск, предпросмотр медиа).
3.3. API меню: Создание эндпоинтов выдачи меню (с пагинацией для видео-ленты) и фильтрацией по тегам/категориям.  

Эпик 4: Афиша и Мероприятия
4.1. Модели мероприятий: Создание моделей Event и News.  
4.2. API Афиши: Эндпоинты для списка будущих мероприятий, архива и новостей.  

Эпик 5: Бронирование (Столы и Ивенты)
5.1. Модель бронирования: Модель TableBooking (Пользователь, Имя, Дата, Время, Гости, Комментарий, Статус).
5.2. Модель записи на ивент: Модель EventReservation (связь с Event и User).
5.3. API бронирования: Эндпоинты для создания заявки пользователем и просмотра истории в профиле.  
5.4. Права в админке: Настройка группы "Менеджер зала" с доступом только к обработке броней.  

Эпик 6: Информационный контент
6.1. Модели контента: Модель RestaurantInfo (Singleton-модель) для хранения адреса, часов работы, ссылок на 3D-тур.  
6.2. API контента: Эндпоинт отдачи статической информации.

Эпик 7: Фоновые задачи и Push-уведомления
7.1. Интеграция Celery: Настройка Celery + Redis для фоновых задач.
7.2. Интеграция Firebase (FCM): Настройка SDK, добавление поля fcm_token в модель пользователя.
7.3. Сервисные триггеры: Написание логики автоматической отправки Push при изменении статуса брони менеджером.  
7.4. Маркетинговые рассылки: Интерфейс в админке для массовой рассылки Push-уведомлений по сегментам. 

Эпик 8: Интеграция с Flutter (Задачи для мобильного разработчика)
8.1. Базовая настройка: Инициализация Flutter, роутинг (GoRouter), настройка DI и архитектуры сети.
8.2. UI-Кит: Верстка базовых компонентов строго по брендбуку (кнопки, тотемы-иконки, карточки, цвета, шрифты). 
8.3. Экраны без авторизации: Главная, Инфо, Афиша (интеграция API).
8.4. Видео-лента: Интеграция видеоплеера с кэшированием, обработка свайпов (TikTok-style). 
8.5. Авторизация и Профиль: Интеграция экрана ввода номера и SMS, сохранение JWT локально.8.6. Бронирование и Push: Подключение FCM на клиенте, флоу отправки заявки на бронь.

│   ├── urls.py              # Главный роутер (подключение API всех аппок)
│   └── settings/            # Разделенные настройки (из Эпика 1)
│       ├── __init__.py
│       ├── base.py          # Общие настройки
│       ├── dev.py           # Настройки для локальной разработки
│       └── prod.py          # Настройки для продакшена
│
├── apps/                    # Директория с бизнес-логикой
│   ├── __init__.py
│   ├── users/               # Кастомный User, JWT, логика SMS-кодов
│   ├── menu/                # Категории, блюда, теги, ссылки на видео/S3
│   ├── events/              # Мероприятия (Афиша) и новости
│   ├── bookings/            # Заявки на столы и записи на ивенты
│   ├── notifications/       # Логика отправки Push (FCM) и шаблоны
│   └── core/                # Статичная инфа (контакты, часы работы, 3D-тур)
│
└── utils/                   # Общие утилиты для всего проекта
    ├── exceptions.py        # Кастомные обработчики ошибок для API
    ├── pagination.py        # Кастомная пагинация (например, для видео-ленты)
    └── services/            # Внешние интеграции (FCM, SMS-шлюз, S3)

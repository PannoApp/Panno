# Seed-видео для feed-ленты

Сюда складываются готовые видео блюд — они подхватываются командой
`seed_demo_content` и проставляются в `Dish.video_processed` со статусом
`video_status = ready`. Celery / FFmpeg не задействуются — это уже
обработанные ассеты.

## Требования к файлам

- Кодек: H.264 + AAC (mp4 предпочтительно; mov работает на iOS)
- Ориентация: вертикальная 720×1280 (как в TikTok / Reels)
- Длительность: 5-15 секунд (соответствует ТЗ раздел 4.2)
- Размер: желательно < 5 МБ на ролик (легче кэшируется `flutter_cache_manager`)

## Как добавить

1. Положи файл в эту папку, например `plov_on_kazan.mp4`
2. В `seed_demo_content.py` пропиши маппинг:
   ```python
   DISH_VIDEOS = {
       "Плов на казане": "plov_on_kazan.mp4",
       ...
   }
   ```
3. Пересоздай контент:
   ```bash
   docker compose exec backend python manage.py seed_demo_content --force --menu-only
   ```
4. В приложении пересвайпи на вкладку Меню → режим «Путь»

## Не коммитить большие файлы

Видео могут быть тяжёлыми — добавь их в `.gitignore`, если не используются в CI:
текущий `demo_dish.mov` — 111 МБ, это плейсхолдер для локальной разработки.
